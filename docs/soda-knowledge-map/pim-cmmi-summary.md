# CMMI

CMMI（Capability Maturity Model Integration / 能力成熟度モデル統合）は、組織のプロセス能力と成熟度を体系的に評価・改善するためのフレームワークです。米カーネギーメロン大学のソフトウェア工学研究所（SEI）で1990年代に開発され、現在はISACA（CMMI Institute）が管理しています。最新版は2023年4月にリリースされた **CMMI v3.0** です。

## モデルの特徴

CMMIは「組織として何を実践しているか」を **プラクティス領域（Practice Area, PA）** の集合で記述し、その実装度合いを **成熟度レベル（Maturity Level, ML1〜ML5）** または **プラクティスグループレベル（1〜5）単位の能力レベル評定** で表現します。プロセス改善の優先順位、ベンチマーク、調達要件の判断基準として、世界中の組織で利用されています。

CMMIは単一のモデルではなく、8つの「ドメイン」（データ・開発・ピープル・セーフティ・セキュリティ・サービス・供給者・バーチャル）を組み合わせる構造を持ち、これらの領域を統合的に扱える点が特徴です。

## 評価とベンチマーク

CMMIの達成度は **Benchmark Appraisal** などの公式アプレイザル（評価）によって判定されます。アプレイザルの結果はISACAの **PARS（Published Appraisal Results System）** に登録され、調達先の選定や組織間の比較に用いられます。歴史的には米国防総省（DoD）系の大規模調達でも要求されてきましたが、近年はインド・中国を中心とするグローバルなアウトソーシング業界が主な需要層となっています。

## 現代の開発との関係

CMMIは伝統的に重厚なプロセス整備のモデルと見なされてきましたが、v2.0以降は「アジャイル／DevOpsと矛盾しない設計」が明示されました。Benchmark Appraisalで成熟度レベルを競う時代は事実上終わりつつあり、**組織能力を俯瞰するためのリファレンス** として活用することが、現代的なCMMIの使い方と言えます。
