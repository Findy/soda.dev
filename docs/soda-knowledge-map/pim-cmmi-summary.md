# CMMI

CMMI（Capability Maturity Model Integration / 能力成熟度モデル統合）は、組織のプロセス能力と成熟度を体系的に評価・改善するためのフレームワークです。米カーネギーメロン大学のソフトウェア工学研究所（SEI）で1990年代に開発され、現在は ISACA（CMMI Institute）が管理しています。最新版は 2023 年4月にリリースされた **CMMI v3.0** です。

## モデルの特徴

CMMI は「組織として何を実践しているか」をプラクティスエリア（Practice Areas）の集合で記述し、その実装度合いを **成熟度レベル（Maturity Level, ML1〜ML5）** または **ケイパビリティレベル（Capability Level, CL0〜CL3）** で表現します。プロセス改善の優先順位、ベンチマーク、調達要件の判断基準として、世界中の組織で利用されています。

CMMI は単一のモデルではなく、複数の「ドメイン／ビュー」（Development、Services、Supplier Management など）を組み合わせる構造を持ち、開発・運用・委託管理・人材・セキュリティ・セーフティといった領域を統合的に扱える点が特徴です。

## 評価とベンチマーク

CMMI の達成度は **Benchmark Appraisal** などの公式アプレイザル（評価）によって判定されます。アプレイザルの結果は ISACA のデータベースに登録され、調達先の選定や組織間の比較に用いられます。特に米国防総省（DoD）系の大規模プロジェクトや、グローバルなアウトソーシング業界で広く要求されています。

## 現代の開発との関係

CMMI は伝統的に重厚なプロセス整備のモデルと見なされてきましたが、v2.0 以降は「アジャイル／DevOps と矛盾しない設計」が明示され、価値の継続的なデリバリーを支えるプロセス能力の枠組みとして再定義されています。
